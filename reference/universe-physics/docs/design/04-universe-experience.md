# 04 — 宇宙板块跨尺度浏览体验设计 (核心)

> 这是整站差异化的根。其他文档可以晚做，这份必须先想清楚。

## 1. 心智模型：尺度阶梯 (Scale Ladder)

把整段下钻拆成**离散的"档"**，每档对应一个尺度等级 (Tier)。档与档之间用相机/场景动画衔接。

| Tier | 名称                 | 量级 (m) | 主体                            | 典型对象密度             |
| ---- | -------------------- | -------- | ------------------------------- | ------------------------ |
| T0   | Observable Universe  | 10²⁶     | 可见宇宙球壳                    | 大尺度纤维               |
| T1   | Cosmic Web           | 10²⁴     | 纤维与空洞                      | 超星系团节点             |
| T2   | Supercluster         | 10²³     | Laniakea / Perseus–Pisces / ... | 星系团 ~ 100             |
| T3   | Cluster / Group      | 10²²     | 本星系群、Virgo Cluster         | 星系 ~ 10–1000           |
| T4   | Galaxy               | 10²¹     | 银河系 / Andromeda              | 恒星 ~ 10¹¹ (点云)       |
| T5   | Stellar Neighborhood | 10¹⁷     | 太阳邻域 ~ 50 ly                | 邻近恒星 ~ 100           |
| T6   | Solar System         | 10¹³（含奥尔特云则 10¹⁵） | 太阳系                          | 行星 / 矮行星 / 主要卫星 |
| T7   | Planet               | 10⁷      | 单个行星 / 卫星                 | 表面 / 大气 / 内部       |

档之间跨度有时跨多个数量级 (T4→T5 跨 ~10⁴)，相机不能"线性飞过去"——必须**对数尺度飞行 + 场景换装**。

## 2. 三种过场策略

### 2.1 Zoom-through (放大穿过) — 跨度 < 100 倍

相机直线推进，目标对象逐渐放大。整个场景资源不变。
典型：T3 → T4 (从本星系群到银河系)、T6 → T7 (太阳系到地球)。

### 2.2 Dissolve-swap (融解换景) — 跨度 100–10000 倍

两个尺度的场景同时存在 ~500ms，旧场景淡出 + 新场景淡入，相机视角延续。
典型：T1 → T2 (纤维结构 → 超星系团)、T4 → T5 (银河系 → 太阳邻域)。

### 2.3 Tunnel (隧道穿越) — 跨度 > 10000 倍

插入一段"虚拟飞行"过场：高速星流条带 + 尺度刻度滚动，期间用户**看不到具体场景**，只感受到尺度在变。≤ 1.5s。
典型：T0 → T1 (可见宇宙 → 纤维结构)、T5 → T6。

> 三选一是体验感的关键。同一对场景永远用同一种过场，不混搭。

## 3. 相机系统

每个 Tier 有自己的"相机舒适区"：

```
舒适距离 (相机离主体)
T0: 半径 1.0–1.5  (相对单位)
T1: 半径 1.5–2.5
T2: 半径 1.2–2.0
T3: 半径 1.5–3.0
T4: 半径 2.0–4.0  (银河系直径外几倍)
T5: 半径 ~30 ly
T6: 海王星轨道外 1.5x
T7: 行星半径 1.5–3x
```

每个 Tier 的 Stage 用**自己的"局部单位"**渲染 (例如太阳系内部用 AU)，避免浮点精度问题。Tier 切换时不是真的把相机飞过 10²⁰ 米，而是**销毁旧场景，加载新场景，相机重置到新 Tier 的舒适区**——但用户感觉是连续的，因为过场动画把这件事**藏起来**了。

技术细节见 `docs/develop/04-animation-strategy.md`。

## 4. 用户在每一层会看到什么

每个 Tier 页面有四个固定要素：

1. **Stage**：该尺度的可视化。
2. **当前主体 (Subject)**：高亮的核心对象 (例如 Laniakea)。
3. **可下钻对象 (Drillable)**：可以点进去的子对象，光标悬停时高亮。
4. **环境对象 (Ambient)**：背景里的东西，给上下文，但不可交互。

例：在 T2 / Laniakea 页：

- Subject：Laniakea 整体形状 + Great Attractor 标记。
- Drillable：本星系群、其他主要群/团 (Virgo, Hydra, Centaurus, ...)。
- Ambient：邻近的 Perseus–Pisces、Shapley 等超星系团的方向标，淡。

## 5. 知识面板的层次

每个 Tier 的知识面板分三层：

1. **关键数据卡** — 一眼就能看到的硬数据 (直径、质量、距离、估算年龄)。
2. **结构解读** — 一段叙事文字 + 一张示意图，解释"为什么它长这样"。
3. **延伸阅读** — 链接到子对象、相关 Tier、外部权威资源 (NASA, ESA, arXiv, Wikipedia)。

不要堆术语词典，那是 Wikipedia 的活。

## 6. 时间维度 (可选 Phase 2)

宇宙不是静态的。Phase 1 用现今快照；Phase 2 可以加一条时间轴：

- 大尺度：宇宙膨胀回溯 (z = 0 → 0.5 → 1 → CMB)。
- 太阳系：行星位置随日期变动。

但**Phase 1 不要加**。先把空间维度做好。

## 7. 入站与首屏

用户从 `/` 进入 → 一段 5–7 秒的入站动画 (可跳过)：

```
黑屏 → 单个白点淡入 → 镜头拉远，白点变成无数点的纤维结构
→ 镜头继续拉远 → 整个可见宇宙球缓缓旋转
→ HUD 淡入，定格在 T0
```

跳过按钮在 1 秒后出现在右下角，文案 "跳过 · Skip"。

跳过 / 看完后，用户落在 `/universe/observable`，即 T0 主页面。
