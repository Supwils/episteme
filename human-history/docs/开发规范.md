# 开发规范

## 为什么 (Why)

统一的开发规范确保代码一致性、可维护性和团队协作效率。本项目是一个纯前端知识图谱网站，使用原生JS + GSAP构建，需要严格的规范来保证动画性能、代码质量和长期可维护性。项目级工程原则见 [工程守则](./工程守则.md)。

## 语言规则

| 场景 | 语言 |
|------|------|
| 所有 `.md` 文档文件 | **中文** |
| 与用户的对话 | **中文** |
| 代码、shell命令、配置文件 | **英文** |
| 文件名、目录名、变量名、函数名 | **英文** |
| 文档中的API/库/框架名称 | 保持英文原文 |

## 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 构建工具 | Vite | ^8.0 |
| 语言 | JavaScript (ES Modules) | ES2022 |
| 动画库 | GSAP (GreenSock) | ^3.12 |
| 样式 | 原生CSS + CSS Custom Properties | - |
| 包管理 | npm | - |
| 路由 | Browser History Router (自研) | - |

**禁止引入**：React、Vue、Angular、TypeScript、Tailwind CSS、D3.js。本项目坚持原生方案。

## 如何应用 (How to apply)

### 文件大小限制

- **JS文件**：单文件 ≤ 300行（软限制），≤ 600行（硬限制）
- **CSS文件**：单文件 ≤ 400行（软限制）
- **MD文档**：单文件 ≤ 500行（软限制），≤ 1000行（硬限制）
- **函数**：≤ 50行（硬限制），≤ 30行（推荐）

### 命名规范

- 文件名：`kebab-case`（如 `timeline.js`, `main.css`）
- 变量/函数：`camelCase`（如 `activeEra`, `formatYear`）
- 常量：`UPPER_SNAKE_CASE`（如 `ERAS`, `CAT_LABELS`）
- CSS类名：`kebab-case`（如 `.era-card`, `.timeline-track`）
- **禁止**：`*Manager`, `*Helper`, `*Util`, `*Service` 等模糊命名

### 模块组织

- 按功能（页面）组织，不按层级
- 每个页面一个文件，导出一个 `render*` 函数
- 共享逻辑放入 `lib/`，禁止把具体页面业务判断塞进通用工具
- 数据层与视图层分离
- 页面状态、事件绑定、动画循环和清理函数必须在同一页面模块内闭环

### CSS规范

- 使用CSS Custom Properties作为设计令牌
- 所有颜色通过 `var(--xxx)` 引用
- 响应式使用 `@media` 断点
- 动画优先使用GSAP，CSS仅用于简单过渡
- 必须支持 `prefers-reduced-motion` 媒体查询

### 动画规范

- 所有动画必须检查 `prefers-reduced-motion`
- 减少动效模式下跳过初始状态，直接显示最终态
- 使用GSAP的 `gsap.fromTo` / `gsap.from` 控制入场动画
- 滚动触发动画使用 `IntersectionObserver`
- Canvas动画必须有 `cancelAnimationFrame` 清理

### 数据规范

- 基础历史数据集中在 `src/data/index.js`
- Atlas分层内容集中在 `src/data/atlas-content.js`
- 新增数据条目时更新对应的 `ERAS`, `EVENTS`, `FIGURES`, `REGIONS` 数组
- 每个事件必须包含：`year`, `title`, `desc`, `era`, `region`, `cat`
- 每个人物必须包含：`name`, `birth`, `death`, `title`, `desc`, `era`, `region`

### 路由规范

- 使用History路由：`/`, `/timeline`, `/atlas`, `/graph`, `/map`, `/figures`
- 新增页面：在 `src/pages/` 创建文件，在 `main.js` 注册路由
- 每次路由切换清空 `#app` 并重新渲染
- 路由生命周期顺序必须是：清理旧页面副作用 → 更新导航状态 → 渲染新页面

### 调试规范

- 先用浏览器或命令复现，再改代码
- 涉及页面交互时，必须验证至少一条完整用户路径
- 涉及全局事件、Canvas动画、定时器或 `requestAnimationFrame` 时，必须验证清理逻辑
- 修改完成后至少运行 `npm run build`

## 反例 (Anti-patterns)

### 禁止：在页面文件中直接操作DOM而不清理

```js
// ❌ 错误：不清理旧内容
export function renderTimeline() {
  const div = document.createElement('div');
  document.getElementById('app').appendChild(div);
}

// ✅ 正确：先清空再渲染
export function renderTimeline() {
  const app = clearApp();
  const div = document.createElement('div');
  app.appendChild(div);
}
```

### 禁止：忽略动效偏好

```js
// ❌ 错误：无条件执行动画
gsap.from(el, { opacity: 0, y: 30 });

// ✅ 正确：检查偏好
if (!prefersReducedMotion()) {
  gsap.from(el, { opacity: 0, y: 30 });
}
```

### 禁止：在循环中创建闭包导致内存泄漏

```js
// ❌ 错误：事件监听器未清理
for (const item of items) {
  canvas.addEventListener('click', () => { ... });
}

// ✅ 正确：使用事件委托
canvas.addEventListener('click', (e) => {
  const item = e.target.closest('.item');
  if (item) { ... }
});
```

## 检查清单 (Checklist)

- [ ] 文件名使用 `kebab-case`
- [ ] 单个JS文件不超过300行
- [ ] 单个函数不超过50行
- [ ] 所有动画检查 `prefers-reduced-motion`
- [ ] Canvas动画有清理逻辑（`cancelAnimationFrame`）
- [ ] 路由切换时先 `clearApp()`
- [ ] 颜色使用CSS变量而非硬编码
- [ ] 新增数据条目更新所有相关数组
- [ ] 响应式布局在移动端正常显示
- [ ] 文档使用中文撰写
