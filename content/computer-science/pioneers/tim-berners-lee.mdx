# 蒂姆·伯纳斯-李

1989 年 3 月，一位在瑞士日内瓦欧洲核子研究中心（CERN）工作的英国工程师向老板提交了一份建议书，标题是《信息管理：一个提案》（*Information Management: A Proposal*）。老板在封面上写了四个字："模糊但令人兴奋。"这份"模糊的提案"，成为万维网（World Wide Web）的起点。这个人叫蒂姆·伯纳斯-李（Tim Berners-Lee）。

## 破除误解：万维网不等于互联网

这是最常见的混淆。互联网（Internet）是一套全球物理和逻辑网络基础设施，由 TCP/IP 协议栈支撑，其前身 ARPANET 早在 1969 年就存在，并非伯纳斯-李的发明。

万维网（WWW）是在互联网上运行的一个**应用层系统**：以 URL（统一资源定位符）命名资源，以 HTTP（超文本传输协议）传输，以 HTML（超文本标记语言）描述内容，并通过超链接（hyperlinks）把分散的文档连接成网。没有互联网，万维网无处运行；有了互联网，万维网才让普通人第一次真正进入它。

伯纳斯-李发明的是这一应用层的完整架构，以及使其成为开放标准的一系列决定。

## 现场：CERN 的文档噩梦

伯纳斯-李 1955 年 6 月 8 日生于伦敦，父母都是数学家，参与了英国早期计算机 Ferranti Mark 1 的工作。他在牛津大学获得物理学学士学位（1976 年）。

1980 年，他作为独立顾问在 CERN 工作了六个月，期间写了一个内部程序 ENQUIRE，用于在项目文档和人员之间建立随机关联——这是万维网思想的早期雏形。

1984 年他以正式员工身份回到 CERN。CERN 有数千名研究人员、数十台不兼容的计算机系统，文档管理是严重问题：人员流动导致制度性知识不断流失，新人花大量时间寻找已有的信息。

1989 年的那份提案，正是针对这个具体问题的工程解答。

## 核心：三个协议，一套开放架构

万维网的技术核心是三个层次：

**URL（统一资源定位符）**：给互联网上每一个资源一个唯一地址。格式 `scheme://host/path` 直到今天几乎未变。

**HTTP（超文本传输协议）**：客户端与服务器之间的请求-响应协议。最初极为简单（HTTP/0.9 只有 GET 方法），后经 HTTP/1.1、HTTP/2、HTTP/3（基于 QUIC）不断演进，但基本请求-响应模型延续至今。

**HTML（超文本标记语言）**：描述网页结构与内容的标记语言，从 SGML 演化而来。最关键的元素是 `<a href="...">` 超链接——它使文档之间的连接成为可能，而不仅仅是单个孤立文档的展示。

1990 年 12 月，第一个网站在 CERN 上线，也是世界上第一个 Web 服务器。1991 年，伯纳斯-李将 Web 发布到互联网讨论组，公开可用。

## 关键决定：拒绝专利

1993 年，CERN 决定将万维网技术的核心规范**公开为免版税标准**——任何人都可以免费使用 HTTP、URL、HTML 建设网站，无需授权费用。

这个决定并非必然。如果 Web 技术被专利保护，每个网站都要向 CERN 或伯纳斯-李付费——Web 或许会成为一项商业服务，而非开放的全球基础设施。

伯纳斯-李本人也拒绝了将万维网商业化的诸多机会，坚持它应该是开放的。他本可以成为比尔·盖茨式的富翁——他选择了不这么做。这是技术史上少见的、个人意志影响技术开放性的案例。

## W3C 与标准化战争

1994 年，伯纳斯-李在 MIT 创立了万维网联盟（W3C），作为 Web 标准的协调机构。此后的历史并非总是顺利：

1990 年代的"浏览器战争"（Netscape vs Internet Explorer）期间，各厂商争相添加私有 HTML 扩展，网页作者不得不为多个浏览器写不同代码。W3C 推进标准的速度跟不上市场变化。

2000 年代，W3C 推进 XHTML 2.0（要求更严格的 XML 格式），遭到浏览器厂商强烈抵制——他们认为这脱离现实。最终，由 Mozilla、Apple、Opera 联合另立 WHATWG 组织，推进更实用的 HTML5 标准，W3C 后来承认 WHATWG 的版本为事实标准。这段历史表明，标准制定是技术、商业、政治利益的复杂博弈，没有单一权威。

## 晚年的担忧：Web 的异化

伯纳斯-李对当今 Web 的走向深感忧虑。他的核心批评是：

- **中心化**：原本去中心化的 Web 被少数巨型平台（搜索、社交媒体）主导，形成新的权力集中。
- **监控与隐私**：用户数据被大规模采集和变现，用户对自己数据的控制权极弱。
- **虚假信息**：平台的推荐算法优化参与度而非真实性，助长了有害内容的传播。

为此，他推进 Solid 项目（Social Linked Data）——一个旨在让用户控制自己数据的去中心化架构。Solid 的实际影响力至今有限，能否撼动现有格局，学界和业界都持怀疑态度。

2016 年，他获得 ACM 图灵奖，表彰他发明万维网、第一个浏览器，以及让 Web 得以扩展的基本协议与算法。

## 超文本的思想史前驱

伯纳斯-李不是凭空发明超链接的。他的工作建立在两位思想先驱的基础上：

**万尼瓦尔·布什（Vannevar Bush）**：1945 年在《大西洋月刊》发表《诚如所思》（*As We May Think*），设想了"Memex"——一台用微缩胶卷存储书籍和记录、并能通过"关联链接"把文献关联起来的假想设备。这是超文本概念的最早系统阐述，早于任何数字计算机的超文本实现。

**特德·纳尔逊（Ted Nelson）**：1960 年代提出了"超文本"（hypertext）这个词，并设计了雄心勃勃的"Xanadu"项目——一个全球文档链接系统，支持双向链接和版权追踪。Xanadu 开发了数十年，从未作为可用系统交付，但它的部分思想（双向链接、内容版权）在哲学上比 Web 更完整。

伯纳斯-李的 Web 之所以成功而 Xanadu 未能，一个关键原因是：Web 接受了不完美——允许死链接、单向链接——把可用性放在了完整性之前。这是工程上的务实主义战胜了理论上的完美主义的一个典型案例。

## Web 标准的实际运作：W3C 内部的政治

W3C 的标准化过程比外部观察者通常认为的复杂得多，它是一个多方博弈的政治过程：

W3C 的成员包括浏览器厂商（Mozilla、Apple、Google、Microsoft）、操作系统公司、内容提供商、学术机构等，各方对标准的利益诉求不同甚至相互冲突。浏览器厂商希望标准稳定（避免大规模重构）同时有足够空间进行差异化；内容提供商希望强大的 API（JavaScript、CSS 动画、WebAssembly）；隐私倡导者希望限制追踪技术；广告行业希望保留定向广告的技术基础。

这些博弈中最典型的近年案例是"第三方 Cookie 退出"争论：Google 提出 2022 年在 Chrome 中弃用第三方 Cookie（追踪广告的核心机制），提出了"隐私沙箱"（Privacy Sandbox）作为替代，但广告行业、出版商和竞争监管机构（英国竞争与市场管理局，CMA）对其替代方案的垄断风险提出质疑，导致这一计划多次推迟，至 2024 年 Google 宣布放弃弃用计划，转而给用户选择权。

这说明，Web 标准不只是技术文档，它是控制互联网权力结构的关键战场。

## 语义网：未兑现的下一代 Web

2001 年，伯纳斯-李在《科学美国人》上发表文章，描述了他对"语义网"（Semantic Web）的愿景：Web 上的数据不只是给人读的，还要有机器可理解的含义，通过 RDF、OWL 等标准让计算机能自动推理不同网站上的相关信息。

二十多年过去了，语义网并未按照原来的设想普及。部分技术（schema.org 的结构化数据、知识图谱）被搜索引擎大量采用，但 Berners-Lee 最初设想的全面语义推理并未实现。

与此同时，知识图谱（Google Knowledge Graph、Wikidata）和大型语言模型在某种意义上以完全不同的方式实现了部分语义理解——不是通过显式本体，而是通过从海量文本中学习的隐式表示。这是否意味着语义网的方向从根本上是错的，或者只是时机太早，技术栈选择不当？这个问题至今有争议。

## 跨域连接

- **网络科学**：万维网本身是迄今被研究得最深入的真实网络之一——其链接结构、幂律分布（少数节点拥有海量链接）、小世界特性，推动了 21 世纪网络科学的兴起。
- **信息哲学**：Web 将"发布"的权力交给了任何人，打破了传统媒体对信息流通的垄断——这既是民主化，也带来了新的混乱。
- **密码学**：HTTPS（HTTP + TLS）的普及，将网络通信的加密从例外变成默认，参见 [[alan-turing]] 中的密码学脉络。

## Web 的经济结构：免费背后的代价

万维网对个人用户是免费的——这是它扩张速度的关键因素，也是它在商业模式上走向今天格局的深层原因。

当"内容免费"成为用户期待的默认值后，支撑内容生产的经济基础必须从其他地方来。广告成为了主要答案。这进而形成了"注意力经济"的逻辑：平台的收入来源于广告，广告的价值来源于用户注意力，因此平台被激励最大化用户在平台上的停留时间，而非最大化用户获取的信息质量。

这个逻辑并非伯纳斯-李设计万维网时的初衷，但它是 Web 开放架构与市场经济的特定组合所产生的系统性后果。修改这个结构——如欧盟的《数字市场法》（DMA）、《数字服务法》（DSA）尝试做的——需要在法律和监管层面上重构激励机制，技术本身无法单独解决。

## Solid 项目：能否重建去中心化 Web

伯纳斯-李 2016 年后主导的 Solid 项目试图从技术层面重新分权：

**核心思路**：用户在自己控制的"Pod"（Personal Online Datastore）中存储所有数据，应用程序向 Pod 请求数据访问权限，而非拥有数据本身。这将数据控制权从平台转移回个人。

**面对的挑战**：现有平台的网络效应极强——人们在某个平台上有朋友、有历史记录、有推荐算法建立的偏好模型。技术上优雅的去中心化替代方案，如果没有足够多的人同时迁移，就无法提供同等的社交价值。

**现状评估**：截至 2026 年，Solid 在技术社区中有一批积极支持者，但主流用户采用仍然非常有限。这与 ActivityPub（Mastodon 等去中心化社交网络使用的协议）的境遇类似——技术上可行，但对抗网络效应的惰性极其困难。

伯纳斯-李的 Solid 项目是否能真正撼动中心化格局，或者只是一个小众替代方案，是一个开放的、答案仍在展开中的问题。
