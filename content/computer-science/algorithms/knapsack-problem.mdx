# 背包问题

一位探险家准备出发，背包最多承重 $W$ 公斤。他面前有 $n$ 件宝物，每件有重量 $w_i$ 和价值 $v_i$。他应该带哪些宝物，才能让总价值最大，同时不超过背包承重？

这就是著名的**0-1 背包问题（0-1 Knapsack Problem）**——"0-1"意味着每件物品要么完整带走（1），要么完全不带（0），不能只带一部分。

看起来简单，但它是 NP 完全问题。理解背包问题，就是理解为什么有些优化问题在计算上是"困难的"，以及动态规划如何在实践中给出精确解。

## 破除误解：贪心无法解 0-1 背包

直觉上，按"价值密度"（$v_i / w_i$，每公斤价值）从高到低贪心选取，似乎合理。但这对 0-1 背包是错的。

**反例**：背包容量 $W = 10$

| 物品 | 重量 | 价值 | 密度 |
|------|------|------|------|
| A    | 6    | 12   | 2.0  |
| B    | 4    | 7    | 1.75 |
| C    | 4    | 7    | 1.75 |

贪心：先选 A（密度最高），剩余容量 4，选 B，总价值 19。
最优：选 B + C，总价值 14……等等，这里最优是选 A+B=19 或 B+C=14。

调整反例：$W=7$，物品 A(重 4，价值 3)，B(重 3，价值 2)，C(重 5，价值 4)：
- 密度：A=0.75, B=0.67, C=0.8，贪心选 C（密度最高），再选 B（剩余容量2放不下B=3），总价值 4
- 最优：A + B = 5 > 4

贪心对**分数背包**（可以按比例取物品的一部分）给出精确最优解，但对 0-1 背包无效。

## 动态规划解法

**状态定义**：$dp[i][j]$ = 考虑前 $i$ 件物品，背包容量为 $j$ 时的最大价值。

**递推关系**：

$$dp[i][j] = \begin{cases} dp[i-1][j] & \text{若 } w_i > j \text{（第 } i \text{ 件放不下）} \\ \max(dp[i-1][j],\ dp[i-1][j-w_i] + v_i) & \text{若 } w_i \leq j \end{cases}$$

直觉：第 $i$ 件物品的选择只有两种——不选（沿用前 $i-1$ 件的最优解），或选（前 $i-1$ 件在容量 $j - w_i$ 下的最优值 + 第 $i$ 件价值）。

**时间复杂度**：$O(nW)$（填充 $n \times W$ 的二维表格）。

**空间优化**：由于 $dp[i]$ 只依赖 $dp[i-1]$，可用一维数组滚动更新，空间降至 $O(W)$：

```python
dp = [0] * (W + 1)
for i in range(n):
    for j in range(W, w[i] - 1, -1):  # 从大到小，防止重复使用
        dp[j] = max(dp[j], dp[j - w[i]] + v[i])
```

注意：**从大到小遍历容量**是关键——若从小到大，同一件物品可能被多次使用（变成"无限背包"）。

## 背包变种

### 完全背包（Unbounded Knapsack）

每件物品可以取任意多件。只需将 $j$ 的遍历方向改为**从小到大**：

```python
for i in range(n):
    for j in range(w[i], W + 1):  # 从小到大！
        dp[j] = max(dp[j], dp[j - w[i]] + v[i])
```

### 有限背包（Bounded Knapsack）

每件物品 $i$ 最多取 $c_i$ 次。朴素方法：展开为 $c_i$ 件相同物品的 0-1 背包，$O(n \cdot c \cdot W)$（$c$ 为平均数量）。

优化：**二进制分组**——将 $c_i$ 件物品分组为 $1, 2, 4, \ldots$ 件（二进制表示），每组视为一件新物品，做 0-1 背包，复杂度降至 $O(n \log c \cdot W)$。

### 多维背包

背包有多个容量限制（如重量和体积），每件物品有对应的多个消耗。DP 扩展为多维状态 $dp[i][j_1][j_2]$，时间 $O(n \cdot W_1 \cdot W_2)$。

### 分组背包

物品分为若干组，每组最多选一件：

```python
for k in groups:
    for j in range(W, -1, -1):
        for i in group_k:
            if j >= w[i]:
                dp[j] = max(dp[j], dp[j - w[i]] + v[i])
```

### 恰好装满 vs 不超过

有时要求"恰好装满"（而非"不超过"），初始化不同：
- 不超过：`dp[0] = 0`，其余 $0$（允许不满）
- 恰好装满：`dp[0] = 0`，其余 $-\infty$（未能恰好装满时视为无效）

## 为什么 0-1 背包是 NP 完全的？

注意：$O(nW)$ 的 DP 看起来是多项式时间，为什么 0-1 背包是 NP 完全的？

**伪多项式时间（Pseudo-polynomial Time）**：$O(nW)$ 是关于 $n$ 和 $W$ 的多项式，但**不是关于输入编码长度的多项式**。

输入编码长度约为 $O(n \log W)$（用二进制表示 $W$ 需要 $\log W$ 位）。因此 $W$ 相对于输入大小是指数级的（$W = 2^{\log W}$），$O(nW)$ 实际上是指数时间。

当 $W$ 数值很大时（如 $W = 10^9$），即使 $n$ 很小，$O(nW)$ 也无法实际运行。这是 NP 完全性的本质：**没有关于输入长度的多项式时间算法**（除非 P=NP）。

然而，当 $W$ 数值"合理"时（如 $W \leq 10^6$），DP 是完全可行的精确算法。这正是背包问题在实践中被广泛应用的原因。

## FPTAS：任意精度近似

若 $W$ 过大，可以用**完全多项式时间近似方案（FPTAS）**：对物品价值做缩放（除以一个因子并取整），将价值变小后用 DP 求解，再映射回原始价值。

给定误差参数 $\varepsilon > 0$，FPTAS 在 $O(n^2 / \varepsilon)$ 时间内给出 $(1-\varepsilon)$ 近似解——价值至少是最优解的 $1-\varepsilon$ 倍。

## 子集和问题：背包的特殊情形

**子集和（Subset Sum）**：给定集合 $S$ 和目标值 $T$，是否存在子集使和恰为 $T$？

这是背包问题 $v_i = w_i, W = T$ 时的判定版本，同样 NP 完全，同样有伪多项式时间 DP 解 $O(nT)$。

子集和是密码学中"背包密码（Knapsack Cryptosystem）"的基础——Merkle 和 Hellman 在 1978 年提出，但随后被 Shamir 在 1984 年用格基规约方法破解，成为密码学历史上著名的失败案例。

## 跨域连接

- **[[dynamic-programming]]**：背包问题是动态规划教学的经典案例，完整展示了状态定义、递推关系、边界条件和时间空间权衡的全过程。
- **[[greedy-algorithms]]**：分数背包是贪心的精确应用；0-1 背包展示了贪心的失效边界——同一类问题的两个变种，一个贪心可行，一个不可行。
- **[[backtracking]]**：回溯可以精确求解背包问题（枚举所有子集），是 DP 解的指数时间替代。理解何时用 DP、何时必须回溯是算法选择的核心判断。
- **运筹学（整数规划）**：0-1 背包是整数线性规划（ILP）的特殊情形，工业界用分支定界法（Branch and Bound）求解大规模实例——结合了 LP 松弛、回溯和界的估算。
