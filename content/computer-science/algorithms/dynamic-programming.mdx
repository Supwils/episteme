# 动态规划

"Dynamic Programming"这个名字，是 Richard Bellman（美国应用数学家）在 1950 年代发明的。他后来在自传里坦承，选这个名字是因为"dynamic"听起来酷，而"programming"在当时意指"规划"（不是写代码），还因为他需要一个让军方资助人"无法反对"的词汇来隐藏他的数学研究。

名字虽然来历诡异，这个思想本身却是算法设计中最强大的范式之一——把看似指数复杂度的问题，通过重用计算结果，降到多项式时间。

## 破除误解：DP 不是一种固定算法

"动态规划"不是一个特定的算法，而是一种**解题范式**：把大问题分解为小问题，把小问题的结果存下来，避免重复计算。

与贪心算法（每步做局部最优选择）不同，动态规划在子问题上**穷举所有可能**，但通过记录已算过的子问题结果，避免指数级的重复。

两个充分条件：
1. **最优子结构**：整体问题的最优解，由子问题的最优解组成
2. **重叠子问题**：不同的分解路径会反复遇到同样的子问题（若无重叠，分治法即可，无需 DP）

## 从斐波那契说起

斐波那契数列：$F(1)=1, F(2)=1, F(n)=F(n-1)+F(n-2)$

朴素递归实现：

```python
def fib(n):
    if n <= 1: return n
    return fib(n-1) + fib(n-2)
```

看起来简洁，实则指数爆炸：计算 $F(5)$ 需要计算 $F(3)$ 两次、$F(2)$ 三次……$F(n)$ 的调用次数是 $O(2^n)$。

原因：大量重复计算。计算树如下：

```
              F(5)
            /      \
          F(4)     F(3)
         /    \    /  \
       F(3)  F(2) F(2) F(1)
       /  \
     F(2) F(1)
```

**记忆化（Memoization）**：把算过的值存起来，再遇到直接返回：

```python
from functools import lru_cache

@lru_cache(maxsize=None)
def fib(n):
    if n <= 1: return n
    return fib(n-1) + fib(n-2)
```

时间复杂度降至 $O(n)$，每个子问题只算一次。

**自底向上（Tabulation）**：不用递归，按子问题依赖顺序迭代：

```python
def fib(n):
    if n <= 1: return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]
```

## 经典问题：最长公共子序列（LCS）

给两个字符串 $X = x_1 x_2 \ldots x_m$ 和 $Y = y_1 y_2 \ldots y_n$，求它们最长的公共子序列（子序列不要求连续，但保持顺序）。

状态定义：$dp[i][j]$ = $X$ 的前 $i$ 个字符和 $Y$ 的前 $j$ 个字符的最长公共子序列长度。

状态转移方程：

$$dp[i][j] = \begin{cases} dp[i-1][j-1] + 1 & \text{if } x_i = y_j \\ \max(dp[i-1][j],\ dp[i][j-1]) & \text{if } x_i \neq y_j \end{cases}$$

```python
def lcs(X, Y):
    m, n = len(X), len(Y)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i-1] == Y[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]
```

时间复杂度：$O(mn)$，空间：$O(mn)$（可优化为 $O(\min(m,n))$）。

LCS 是 `diff`（比较两个文件差异的命令行工具）的核心算法，也用于生物信息学中的 DNA/蛋白质序列比对。

## 背包问题：离散优化的原型

0/1 背包问题：$n$ 件物品，第 $i$ 件重量 $w_i$，价值 $v_i$。背包容量 $W$，每件物品选或不选，求最大总价值。

状态：$dp[i][j]$ = 前 $i$ 件物品在容量 $j$ 下的最大价值。

状态转移：

$$dp[i][j] = \max\left(dp[i-1][j],\ dp[i-1][j-w_i] + v_i\right)$$

第一项：不选第 $i$ 件；第二项：选第 $i$ 件（前提是装得下）。

时间：$O(nW)$，空间：$O(nW)$（可优化为 $O(W)$）。

背包问题是许多实际优化问题的抽象：资源分配、投资组合、任务调度。

## 编辑距离：拼写纠错背后的 DP

**编辑距离（Levenshtein Distance）**：把字符串 $A$ 转化为字符串 $B$ 所需的最少操作数，每次操作可以是插入、删除或替换一个字符。

例如，"kitten" → "sitting"（编辑距离为 3）：
1. kitten → **s**itten（替换 k 为 s）
2. sitt**e**n → sitt**i**n（替换 e 为 i）
3. sittin → sittin**g**（插入 g）

DP 定义：$dp[i][j]$ = 把 $A$ 的前 $i$ 个字符变为 $B$ 的前 $j$ 个字符所需的最少操作数。

$$dp[i][j] = \begin{cases} dp[i-1][j-1] & \text{if } A[i] = B[j] \text{（无需操作）} \\ 1 + \min(dp[i-1][j],\ dp[i][j-1],\ dp[i-1][j-1]) & \text{否则（删/插/替）} \end{cases}$$

时间和空间均为 $O(mn)$，可优化到 $O(\min(m, n))$ 空间。

实际应用：拼写检查器（"你是否想搜索 X？"）、DNA 序列比对（Smith-Waterman 变体）、diff 工具（Git 中两个文件版本的对比）、语音识别的词错误率（WER）计算。

## 另一个视角：DP 与最短路径

动态规划可以看作**有向无环图（DAG）上的最短（或最长）路径问题**：

- 每个"状态"是 DAG 上的一个节点
- "状态转移"是 DAG 的边
- 求最优解等价于求 DAG 上的最短/最长路径

这把 DP 和图算法统一进同一框架。Dijkstra 对一般加权图求最短路径；DP 在 DAG（无环图）上求最短路径，可以线性时间完成（因为 DAG 有拓扑序）。

## 状态设计的艺术

DP 的难点不在代码，在**状态定义**。状态设计得好，转移简单；设计不好，问题难以求解。

一些常见陷阱：
- 状态维度不够：遗漏了区分子问题所需的关键信息
- 状态维度过多：导致时间空间复杂度爆炸

实践中，识别 DP 问题的信号：问题要求"最大/最小/多少种方案"；问题可以分解为较小的**同类**子问题；且子问题之间有**依赖关系**（不是独立的）。

## 代价与争议

**不是万灵药**：DP 要求重叠子问题。如果子问题完全独立（如归并排序），分治已经是最优，DP 带来不了额外好处。

**空间与时间的权衡**：记忆化的代价是内存。$O(mn)$ 的 DP 表格在 $m, n$ 很大时可能需要几 GB 内存。实践中常用"滚动数组"优化，只保留几行而非整张表。

**伪多项式时间**：背包问题的 $O(nW)$ 看起来是多项式，但 $W$ 是物品重量的**数值**，而非输入规模的位数。如果物品重量可以达到 $2^{100}$，$W$ 就是 $2^{100}$，算法的真实时间是指数的。这类复杂度称为"伪多项式时间"，背包问题实际上是 NP 困难的。

## 跨域连接

- **[[recursion]]**：DP 的记忆化版本就是带缓存的递归；理解递归是理解 DP 的前提。
- **[[shortest-path-dijkstra]]**：Bellman-Ford 算法是动态规划在最短路径问题上的应用；Floyd-Warshall（全点对最短路径）是经典 DP。
- **生物信息学**：Smith-Waterman 算法（序列比对）、Viterbi 算法（隐马尔科夫模型的解码）是 DP 在计算生物学的核心应用。
- **机器学习**：强化学习的 Bellman 方程正是动态规划方程——价值函数满足最优子结构，可以用 DP（值迭代、策略迭代）求解。
