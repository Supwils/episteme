# 并查集

一张有十亿用户的社交网络，好友关系不断新增。系统要随时回答："张三和李四，通过朋友的朋友的朋友……能不能连上？"

每来一对询问就跑一次深度优先搜索？一次搜索要遍历整张图（$O(n + m)$）；十亿节点上做几百万次询问，等于把整张网络扫几百万遍——根本跑不动。

并查集（Union-Find，又称 Disjoint Set Union，DSU）把每次"合并两个人"和"查询是否连通"的代价压到**摊销几乎 $O(1)$**：哪怕节点数是十亿，每次操作平均只需常数级别的几步。它的实现简单得令人意外——核心就是一个 `parent` 数组——但背后的分析涉及计算机科学史上最精妙的摊销分析之一（见下文 Tarjan 1975 的结果）。

## 核心接口

并查集支持三种操作：

- `make_set(x)`：将 $x$ 初始化为一个单元素集合
- `find(x)`：返回 $x$ 所在集合的**代表（Representative）**，同一集合内所有元素返回相同代表
- `union(x, y)`：合并 $x$ 和 $y$ 所在的两个集合

利用这三个操作，判断 $a, b$ 是否连通只需：`find(a) == find(b)`。

## 数组实现：从朴素到高效

### 朴素方案：Quick-Find

用数组 `id[]`，`id[i]` 存储节点 $i$ 的集合 ID。

- `find(x)`：返回 `id[x]`，$O(1)$
- `union(x, y)`：将 `y` 集合的所有节点的 `id` 改为 `x` 集合的 `id`，$O(n)$

查询快，合并慢。$m$ 次合并 $O(nm)$。

### 朴素方案：Quick-Union

用数组 `parent[]`，每个节点指向"父节点"，根节点指向自己，根节点是集合代表。

- `find(x)`：沿 `parent` 指针向上直到根，$O(\text{树高})$，最坏 $O(n)$
- `union(x, y)`：找到两者根节点，把一个根指向另一个根，$O(1)$（在找到根之后）

树可能退化成链表——最坏 $O(n)$ 查询。

### 优化一：按秩合并（Union by Rank）

合并时，让小树挂在大树下（而不是随意合并），用 `rank[root]` 记录树高度的上界。

这保证树高度 $O(\log n)$，因此 `find` 变为 $O(\log n)$。

### 优化二：路径压缩（Path Compression）

在 `find` 操作中，访问路径上的每个节点时，**直接把它的父节点设为根**：

```python
def find(x):
    if parent[x] != x:
        parent[x] = find(parent[x])  # 路径压缩（递归版）
    return parent[x]
```

这不改变集合的逻辑结构，只是把树"压扁"，使后续查询更快。

### 两个优化组合：近乎 $O(1)$

同时使用**按秩合并**和**路径压缩**，任意 $m$ 次操作的总时间为：

$$O(m \cdot \alpha(n))$$

其中 $\alpha(n)$ 是 Ackermann 函数的反函数。

Ackermann 函数增长极快（$A(4,4)$ 是一个比可观测宇宙中原子数还大的数），因此它的反函数 $\alpha(n)$ 增长极慢：对任何实际可能的 $n$，$\alpha(n) \leq 4$。

实践中：**并查集的每次操作摊销几乎是 $O(1)$**。

这一结果由 Robert Tarjan（图灵奖得主）在 1975 年证明，是算法理论中摊销分析的里程碑。

## 为什么 $\alpha(n)$ 不是真正的 $O(1)$？

这值得一提：1989 年，Fredman 和 Saks 证明了并查集的下界——任何实现的摊销代价都不能比 $\Omega(\alpha(n))$ 更低。

所以 $O(m \cdot \alpha(n))$ **既是上界也是下界**——并查集在渐近意义上已经最优了。但它不是真正的 $O(1)$：这个"几乎常数"是有理论保证的最优。

## 经典应用

### Kruskal 最小生成树

Kruskal 算法按边权从小到大排序，逐条加边，若边两端不连通（用 `find` 判断），则加入生成树（用 `union` 合并）。

并查集是 Kruskal 高效实现的核心。有了路径压缩+按秩合并，整个算法瓶颈在排序 $O(E \log E)$，而非并查集操作。

### 检测无向图中的环

逐边加入，每次检查 `find(u) == find(v)`——若相等说明成环，否则 `union(u, v)`。

### 网络连通性维护

网络节点逐步相连时（如社交网络、通信网），实时回答"$a$ 和 $b$ 能通信吗？"

### 图像处理：连通区域标记

二值图像中，寻找所有连通的像素区域（Connected Components Labeling），可以用并查集以近线性时间完成。

### 离线 LCA（最近公共祖先）

Tarjan 离线 LCA 算法用并查集以线性时间批量回答所有 LCA 查询——并查集路径压缩在这里发挥关键作用。

## 加权并查集：带权信息的扩展

并查集可以扩展为**加权并查集（Weighted DSU）**，在 `parent` 指针上记录从子节点到父节点的权重（如相对距离、差值）。路径压缩时同步更新权重。

应用：判断图中两点之间是否存在特定关系（如大小关系、模余关系），广泛用于算法竞赛的约束系统问题。

## 可持久化并查集

标准并查集不支持"回滚"（撤销 union 操作）。**可持久化并查集（Persistent DSU）**用可持久化数组（基于主席树）实现，可以在 $O(\log n)$ 内查询"历史上任意时刻的集合状态"。

代价：路径压缩会破坏持久化，所以只能用按秩合并（不用路径压缩），时间 $O(\log n)$ 而非 $O(\alpha(n))$。

## 并查集的局限

并查集支持合并（union），但**不支持分裂（split）**——一旦合并，无法把集合再拆开（不支持 `delete` 操作）。

对于需要分裂的动态连通性问题，需要更复杂的数据结构（如 Link-Cut Tree，也由 Tarjan 提出）。

## 跨域连接

- **[[minimum-spanning-tree]]**：Kruskal 算法是并查集最重要的应用场景，两者的结合是图算法中最简洁高效的组合之一。
- **[[greedy-algorithms]]**：Kruskal 是贪心算法，并查集提供其高效实现；贪心正确性保证和并查集的摊销复杂度分析是两套独立的理论。
- **数学**：$\alpha(n)$（Ackermann 函数的反函数）连接了算法分析与逻辑学/递归论，是计算机科学中极少数与高阶递归函数直接相关的复杂度分析。
