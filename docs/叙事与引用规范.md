# 叙事与引用规范 — Universe Knowledge 内容核心准则

> 本文件是平台所有知识内容的**核心创作准则**，与 `知识精神.md` 互补：`知识精神.md` 讲"我们相信什么"，本文件讲"具体怎么写、怎么标来源、机器怎么把关"。
>
> 一句话：**用最清楚的语言说最有深度的话，每一个判断都站得住、查得到。**

---

## 一、叙事标准：通俗易懂 · 形象 · 准确 · 有深度

这四个词不是口号，是四条可检查的要求。一篇合格的文章四条都要满足。

### 1. 通俗易懂（表达层）

- ✅ 一句话只说一件事。中文长句拆短，**句号后该换段就换段**，不要把十几句话挤在一行/一段里。
- ✅ 专业术语第一次出现时用一句话解释，或给一个日常类比。
- ✅ 假设读者聪明但没有专业背景：不堆术语，也不把人当无知者哄。
- ❌ **句子挤成一行**（`。 **下一句**` 这种用空格把本该分段的句子粘在一起）——这是当前最常见的可读性缺陷，`check-content` 会标 `run-on sentences`。
- ❌ 空洞形容词推销：伟大的、辉煌的、震撼的。让事实自己说话。

### 2. 形象（现场感）

- ✅ 给场景、给数字、给画面：「公元前 480 年，三百斯巴达人守在温泉关」胜过「斯巴达人英勇抵抗」。
- ✅ 用具体类比降低理解门槛，但**类比不能丢失关键精确性**（玻尔对话里那句"我不想把它与神秘主义混为一谈"就是好的自我约束）。
- ❌ 猎奇语气渲染战争/灾难/屠杀。

### 3. 准确（求真）

- ✅ 数字、时间、地点、制度名称必须具体且可查证。
- ✅ 数据标注到机构/论文（NASA、ESA、同行评审期刊），关键量给误差范围与测量方法。
- ✅ 区分"已知事实"与"推测/理论预测"：用"目前观测显示……""目前学界认为……"而不是伪装成定论。
- ✅ 人名、译名核对正确（曾出现"诺德豪斯"被损坏成"诺德haus"一类编码错误，提交前自查中英混排）。
- ❌ 对有争议的史实/科学问题用肯定句断言。

### 4. 有深度（不止于维基摘要）

- ✅ **破除误解**：先点出常见但过浅的理解，再纠正（推荐作为开篇结构，CS/政治学/物理/历史已近乎标配）。
- ✅ **争议意识**：呈现主流解释的同时，主动说"但也有人认为……"，给出学界/史学界的分歧。
- ✅ **反对单一叙事**：非欧洲中心、非单线英雄叙事，给失败者与边缘群体位置。
- ✅ **跨域连接**：把历史事件接到哲学思想、宇宙现象接到科学史。
- ❌ 只堆年份和人名、单句 bullet 罗列贡献而不解释"为什么重要"（economics/psychology 早批的主要短板）。

> 最终判断标准（来自 `知识精神.md`）：读者读完应感到"我理解了以前不懂的东西，而且我相信它是真的""我想了解更多"。如果只是"哦，知道了"然后关页面，就还不够好。

## 二、引用结构标准

可信赖是平台的立身之本。引用既是对读者负责，也是 SEO 与可信度信号（文章页会把书目输出为 schema.org `Article.citation` JSON-LD）。

### 1. 语义分离：参考文献 vs 延伸阅读

| 类别         | 含义                    | 是否计入"来源门槛" |
| ------------ | ----------------------- | ------------------ |
| **参考文献** | 佐证正文事实/数据的来源 | 是                 |
| **延伸阅读** | 推荐但非必需的读物      | 否                 |

历史上两者混用是最大的结构问题。新内容必须区分二者。

### 2. 标准标题（统一）

- ✅ 正文书目统一用 `## 参考文献` 或 `## 延伸阅读`。
- ❌ 不用 `## References`、`## 推荐阅读` 等变体——`check-content` 会标 `non-standard citation heading`。

### 3. 来源质量规则

- ✅ **合格来源**：同行评审论文（带 DOI 更佳）、学术专著（出版社+年份）、权威机构/标准（NASA/NIST/IETF RFC/联合国）、一手史料。
- ⚠️ **可作延伸阅读、但不算佐证来源**：科普畅销书、纪录片。
- ❌ **禁止作为佐证来源**：维基百科 / 百度百科、无出处的"网络资料"、空链接。
- 每条参考文献至少含 `标题` + (`年份` 或 `DOI` 或 `出版社`) 之一。

### 4. 结构化引用（新内容推荐）

新文章可在 frontmatter 用结构化字段（`lib/content-schemas.ts` 的 `CitationSchema`），供未来统一渲染：

```yaml
references: # 佐证来源
  - title: A Comprehensive Measurement of the Local Value of H0
    author: Riess, A. G. et al.
    year: 2022
    publisher: ApJL
    doi: 10.3847/2041-8213/ac5c5b
    type: paper
furtherReading: # 选读
  - title: 宇宙的琴弦
    author: Brian Greene
    type: book
```

存量文章的正文 `## 参考文献` 列表由 `lib/citations.ts::extractBibliography` 自动解析并进 JSON-LD，无需立即迁移。

### 5. 分级要求（按文章类型）

| 文章类型                     | 最低佐证来源        | 备注               |
| ---------------------------- | ------------------- | ------------------ |
| `frontier/*`（研究前沿）     | ≥ 3，且 ≥1 条带 DOI | 最该有引用         |
| `experiments/*`（实验/研究） | ≥ 1 原始论文        | 文件名常即论文本身 |
| 人物 / 理论 / 概念正文       | ≥ 2                 |                    |
| `dialogues/*`（虚构对话体）  | 0（豁免）           | 标注"叙事体"       |

## 三、自动化强制（check-content）

`pnpm check-content` 已对全部内容执行以下校验，**提交前必须本地跑过**：

- **Per-domain frontmatter**（7 个 `.mdx` 域）：必填 `title/status/updated`、Zod schema（有则校验）、篇幅下限、`## 标题`存在、延伸阅读/参考存在、`related` 链接完整性、已发布文件中的 TODO。
- **Link Integrity**：CJK slug 编码往返能否 resolve（防搜索结果 404）。
- **Frontier Articles**：`FrontierSchema` + 篇幅 + 引用。
- **Prose & Citation Hygiene**（全部 `.md` + `.mdx`）：
  - 非标准引用标题 → `non-standard citation heading`
  - 句子挤行 → `run-on sentences crammed on one line`

> 当前为 0 error / 数百 warning。warning 不阻塞构建，但它们是 Phase 3 内容返工的工作清单——目标是把 warning 持续降到 0。

### 提交前自查清单

1. `pnpm check-content` —— 0 error，留意自己改的文件是否新增 warning。
2. 四条叙事标准过一遍：是否有破除误解 / 争议 / 跨域 / 具体数字与出处？
3. 书目用了 `## 参考文献` / `## 延伸阅读`？来源是真实出版物而非维基百科？
4. 中英混排、人名译名、句子分段是否正确？
