# 内容深化轮指南（10 学科）

> 本轮目标：为每个学科**新增高质量知识** + **改善现有内容的叙事**。每个学科一个代理。

## 两项任务

**任务 A — 新增内容**：补该学科的明显空白，写 web 核实的新文章。
**任务 B — 改善叙事**：挑该学科现有 3-5 篇较薄或平淡的文章，深化与打磨（补机制、数据、争议、跨域连接、来源），不破坏 frontmatter。

## 如何新增（按内容类型）

动笔前先 `ls` 目标目录并**读 2-3 个现有文件**，照抄其 frontmatter 形状与 slug 风格。

- **类型化 MDX 学科**（philosophy / economics / psychology / mathematics）：新文章写 `content/<域>/<子类>/<slug>.mdx`，并**必须**在该子类的 `content/<域>/<子类>-data.ts`（或对应 data 文件）追加一条 `{ slug, title, subtitle/titleEn, ... }`，形状与既有条目完全一致——否则新页不会进列表与搜索。
- **知识库学科**（universe-physics / cosmology / human-history）：新文章写 `content/<域>/knowledge-base/<slug>.md`（目录自动成路由），并在 `content/<域>/knowledge-base-data.ts` 追加条目用于搜索。
- **生命科学**：新增 scientists/species/events 到对应目录，照现有 frontmatter；若有索引文件（`content/life-science/index.ts` 或 `subjects/life-science/lib`）需同步。
- **新领域**（computer-science / political-science）：新文章写 `content/<域>/<板块>/<slug>.mdx`，frontmatter 见 `docs/新领域创作指南.md`；**无需** data 文件（编排者会用 `pnpm gen-domain` 自动生成搜索数据）。

## 写作标准（遵循 docs/知识精神.md）

- "顶级教授讲课"式：有现场感、有结构、有争议意识、有跨域连接（可用 `[[slug]]`）。
- 每篇 ≥ 100 非空行（人物/重大主题）或 ≥ 80 行（一般条目）；段落不要写成长墙。
- 结尾有真实"延伸阅读/参考文献"。

## 事实纪律（最高优先级）

- 凡具体年份/数据/人名/论断/引语，用 WebSearch 核实；**绝不编造**人名、论文、机构、数据、名言。不确定就概括，不要造看似具体的"事实"。
- 区分"已确证"与"有争议/推测"；在世/已故必须核实。引语无法溯源就转述或标"传"。
- frontmatter：`status: published`、`updated: 2026-06-13`；含半角冒号 `: ` 的 `title_en`/列表值必须加引号（否则 YAML 解析失败）。

## 硬约束

- 只动**本学科**的 `content/<域>/` 与其 data 文件。不要碰 lib/、app/、components/、scripts/、其他学科。
- 不要运行 `pnpm build`/`dev`/`typecheck`（编排者统一验证）；可运行 `pnpm check-content` 自检。
- 完成后报告：新增文件（slug + 一行摘要）、深化的文件（改了什么）、更新的 data 文件条目、无法核实而省略的内容。
