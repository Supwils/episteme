## 一、执行摘要

**核心结论：这不是一个"内容浅、技术债重"的项目，而是一个"内容优秀、基础设施合格、需要做工程化收尾"的项目。**

发起本次审查的最大担忧是——平台主体由较低智能的模型生成，内容可能是空壳。**这个担忧被证据系统性推翻。** 五个领域的深度抽样精读一致表明：历史、哲学、经济学、心理学、数学、生命科学、宇宙学的核心内容**真实达到本科到研究生水平，有真实数据、一手学术引用、跨领域连接，甚至正确标注了学术争议（如心理学复制危机）**——后者是低能力模型几乎做不到的强信号。

真正的问题集中在三类，且都比"重写内容"更容易解决：

1. **安全与产品宪章越界**（唯一真正紧急）：admin 后台用可绕过的客户端口令，admin API 零鉴权；同时平台实现了自己宪章明令"不做"的功能（书签/阅读进度/分享 analytics）。
2. **内容可达性断裂**（高价值、易修）：写好的优质内容没接到用户面——数学 120 篇 MDX 的公式全部以 `$$` 源码显示、生命科学整个 `content/` 目录和宇宙学 19 篇深度文章不被任何页面渲染、心理学旗舰可视化一半节点点击 404。**"内容白写"是当前最大的价值漏损。**
3. **工程卫生**：重复文件、断链、占位填充、被"固化"成绿色的欺骗性测试、跨领域架构违规、性能预算无机器关卡、遍地大文件。

**一个意外的元发现**：`docs/聚焦方向.md` 列出的六大"已知内容空白"（微观经济学、发展心理学、分析哲学、应用数学、分子生物学、观测天文学）**大部分已经被内容创作填补**——只是部分没接路由。该文档本身已过时，需要更新。

### [P0-1] admin 后台客户端假鉴权 + admin API 零鉴权（安全 + 宪章越界）
- **证据**：`apps/portal/app/admin/stats/page.tsx:119` → `if (key === (process.env.NEXT_PUBLIC_ADMIN_KEY ?? "admin"))`，且整页是 `"use client"` 静态生成。`NEXT_PUBLIC_*` 会被 inline 进浏览器 bundle；`.env.example` 未定义该变量，生产环境 fallback 到字面量 `"admin"`。`apps/portal/app/api/admin/content-stats/route.ts:6` → `export function GET()` 完全无鉴权，内部 `fs.readFileSync` 遍历 `content/`。
- **影响**：`/admin/*` 无任何访问控制；`robots.ts` 的 disallow 只挡爬虫不挡直接访问。当前数据面危害有限（只读聚合统计 + 本机 localStorage 事件，无 PII、无写操作），但"管理后台用可绕过口令"是必须消除的反模式，且它直接违反 CLAUDE.md"平台明确不做用户系统"。
- **修复**：与产品确认后，优选**删除整个 admin 子系统**（含其消费的 analytics 埋点，见 P1-8），把内容统计降级为 dev-only 工具或复用已有的 `pnpm check-content`；若必须保留则上 middleware + 服务端鉴权，密钥放非 `NEXT_PUBLIC_` 变量。
- **工作量**：S（删除）/ M（真鉴权）

### [P0-2] 数学板块 120 篇 MDX 的公式全部以 `$$` 源码显示给用户（整域体验崩塌）

- **影响**：120 篇数学 MDX 中所有公式、定理、积分、矩阵对用户显示为带 `$$` 的原始 TeX 源码。`concepts/[slug]`、`theorems/[slug]`、`paradoxes/[slug]` 全部受影响。内容写得越深，渲染层浪费越严重——这是整个数学领域的核心体验崩塌。
- **修复**：接入 KaTeX（`katex.renderToString`，block 用 `displayMode:true`），或迁移到 `remark-math + rehype-katex`。
- **工作量**：M（约半天接 KaTeX + 样式微调，立刻救活整个数学板块）
