## 一、总体结论：安全姿态 **B+（稳健）**

本站是**无用户登录、无 UGC、内容由维护者撰写**的静态/ISR 知识网站。审计未发现 **critical / high** 漏洞，**当前不存在可达的反射型/存储型 XSS sink**。

正向确认（基线良好，非漏洞）：

- **安全头基线扎实**：`headers()` 用 `/(.*)` 覆盖全路由（含 `/_next`、`/api`、og、daily），全仓唯一 CSP、无 middleware/setHeader 覆盖；`object-src 'none'`、`base-uri 'self'`、`frame-ancestors 'none'`（+ `X-Frame-Options: DENY` 双保险）、`form-action 'self'`、`nosniff`、`Referrer-Policy`、`Permissions-Policy`（关闭 camera/mic/geo/FLoC/topics/payment）、COOP/CORP same-origin 齐备；`'unsafe-eval'` 与 `worker-src blob:` 已用 `isProd` 正确限定为 **dev-only**。
- **正文渲染默认转义**：`MarkdownRenderer` / `MathMarkdownRenderer` 为自研解析器，正文作为 React 字符串子节点输出（自动转义）；`.mdx` 经 `fs.readFileSync` + gray-matter 以纯文本读取，**从不被 @next/mdx 编译为 JSX** → 正文里的 `<script>` 只会作为可见文本显示，不执行。
- **KaTeX 安全**：`renderToString` 未传 `trust`（默认 false），`\href/\url/\includegraphics` 等危险宏禁用；katex 0.17.0 已含 CVE-2025-23207 补丁。
- **OG 路由输出 PNG**：`/api/og` 反射 URL 参数但经 Satori 光栅化为 PNG，无 HTML/JS 执行上下文，非 XSS。

## 二、发现与处置（5 条真实项，均 medium/low · 纵深防御 + 供应链 + 配置硬化）

| #   | 级别   | 标题                                                                                | 状态                                                      |
| --- | ------ | ----------------------------------------------------------------------------------- | --------------------------------------------------------- |
| 1   | medium | 生产 CSP `script-src` 含 `'unsafe-inline'` 且无 nonce/hash                          | ⏳ 待办（需 middleware nonce + 全站浏览器验证，较大工程） |
| 2   | medium | Mol\* 从 jsDelivr 无 SRI 注入 + CSP allowlist 自相矛盾（功能在生产 CSP 下实际已坏） | ✅ 已修复（自托管 + 作用域 CSP，浏览器验证）              |
| 3   | low    | 32 处 JSON-LD 用裸 `JSON.stringify` 注入，未转义 `</script>`                        | ✅ 已修复                                                 |
| 4   | low    | 未显式设 HSTS                                                                       | ✅ 已修复                                                 |
| 5   | low    | 人类历史 vanilla-JS 层 innerHTML 转义不一致                                         | ⏳ 待办（低优先，需 history 页运行时验证）                |

### ✅ #2 Mol\* 自托管 + 作用域 CSP（已浏览器验证）

- **自托管**：molstar 4.9.0 viewer 构建（`molstar.js` 4.8MB / `molstar.css` 74K）下载到 `public/vendor/molstar/`；`MoleculeViewer.tsx` 改为从同源 `/vendor/molstar/*` 注入 → **彻底去掉 jsDelivr 第三方供应链/SRI 面**。
- **作用域 CSP**：molstar 运行时需要 `'unsafe-eval'`（用 new Function/eval + WebAssembly）、RCSB 主机（取真实 PDB）、`data:` connect（取内置 WASM）、`blob:` worker。把这些**仅放行在 `/medicine/*`、`/chemistry/*`、`/molecules/*`**（唯一渲染分子的路由）；catch-all 用负向前看 `/((?!medicine|chemistry|molecules).*)` 保证**全站其余路由保持严格无 eval、无 RCSB、无 data:**。
- **验证**：curl 确认两类路由 CSP 正确分流；Playwright（软件 WebGL）实测分子页 **molstar 全局加载成功、canvas 渲染成功、eval 与 data: 违规清零**；唯一残留是 molstar 默认 `localhost:9000` volume-server 探测被 CSP 正确拦截（无害，仅用 loadPdb）。typecheck + build 全绿。
- **遗留小项**：molstar 的 `localhost:9000` 探测属噪声（功能不受影响）；RCSB 真实结构加载需在生产真实浏览器+网络下做最终冒烟。

### ✅ 已修复（本地验证：typecheck + build 全绿）

- **#3 JSON-LD 转义**：新增 `lib/jsonld.ts::serializeJsonLd()`，转义 `<`/`>`/`&` 防 `</script>` 提前闭合；用 codemod 把全部 **32 处** `__html: JSON.stringify(...)` 统一替换为 `serializeJsonLd(...)`（20 处注入既有 import、12 处新增 import）。同时修复"一条合法含 `</script>` 的引用会破页"的正确性 bug。
- **#4 HSTS**：`next.config.ts` 的 `securityHeaders` 增加 `Strict-Transport-Security: max-age=63072000; includeSubDomains`（**暂不含 `preload`**——它是难撤销的承诺，待确认所有子域 HTTPS 再加）。
- **快速清理**：`connect-src` 去掉无效的重复项 `https://vitals.vercel-insights.com/v1/`（CSP host-source 忽略路径）。

### ⏳ 待办（需决策/测试，影响线上，等用户拍板）

- **#1（最高杠杆）生产 CSP 去 `'unsafe-inline'`**：有 `'unsafe-inline'` 又无 nonce/hash 时浏览器尊重 unsafe-inline，等于关掉 CSP 对脚本注入的兜底。修法：①给 `app/layout.tsx` 几段**静态**内联脚本算 SHA-256 加进 `script-src`；②Next.js 引导脚本则需 **middleware 注入 per-request nonce**。改完必须用浏览器控制台确认零 CSP 违规（建议配合 webapp-testing）。
- **#2 Mol\* 供应链 + allowlist 不一致**：`MoleculeViewer.tsx` 运行时从 `cdn.jsdelivr.net` 注入 molstar.js/css **无 SRI**；而 jsDelivr 只在 `img-src`，不在 script/style/connect-src → **该功能在生产强制 CSP 下其实已被 fail-closed 拦死**。推荐：**自托管 molstar 4.9.0 dist 到 `/public`**（同源、去第三方风险、不进 JS bundle、顺带修好功能）；或保留 CDN 但加 `integrity` SRI + 精确放行 jsDelivr（script/style/connect）与 `models.rcsb.org`/`files.rcsb.org`（connect）。**切勿用 `https:` 通配或加 `unsafe-inline` 草率修**。
- **#5 history innerHTML**：`subjects/history/components/history/*.js` 若干历史数据字段（era.quote/longDesc/legacy、ev.longDesc）直插 innerHTML 未过 `escapeHtml()`。数据为打包进 bundle 的可信文本，当前不可利用；属潜在 stored-XSS 隐患，建议统一套已存在的 `escapeHtml()`。

## 三、已过滤的误报（对抗式排除，留档备查）

- style-src 的 `'unsafe-inline'`：Tailwind/Framer/KaTeX 内联样式所需，风险仅限样式注入，**可接受**。
- "Mol\* 功能被 CSP 拦死"本身不是漏洞——CSP fail-closed 拒绝是好事；已并入 #2 作为功能性 + allowlist 一致性问题。
- 所有 XSS/注入面（unsafe-inline、JSON-LD、history innerHTML）在本架构下**当前均无攻击者可控 sink**（无登录/UGC、内容可信、`[slug]` 先查已存在记录否则 404、URL 参数不原样进 HTML）——故定级为纵深防御而非可利用漏洞。

## 四、下一步建议优先级（更新）

- ✅ **#2 Mol\* 自托管 + 作用域 CSP** — 已完成并浏览器验证。
- ⏳ **#1 CSP nonce**（最高安全杠杆）：用 middleware 注入 per-request nonce 去掉 `script-src` 的 `'unsafe-inline'`，需对全站每类页面做浏览器级 CSP-违规回归，属较大工程，建议专门一轮做。
- ⏳ **#5 history escapeHtml**（廉价硬化）：对 `subjects/history/components/history/*.js` 未转义的数据字段套已存在的 `escapeHtml()`，改后需在 /human-history 页面运行时验证渲染不受影响。
