设计系统交付规范

本文是给工程 agent 的落地依据。它把新的「观测台 / 手记」双主题视觉，对应到 codebase 里真实存在的令牌名、字体与文件——照表替换即可完成迁移，无需逐页重写。

令牌驱动 · 低风险 双主题 10 领域强调色

给 Agent：如何使用本规范

  1. 第 ③ 节「令牌映射」是核心——共享组件（ArticleLayout、MarkdownRenderer）都读这些令牌，改值即可全站生效。
  2. 按第 ⑦ 节「文件清单」逐个文件落地；先改门户 + 一个内容领域验证，再推广。
  3. 定制领域（物理/宇宙/历史/生命科学）保持暗色，只接入新令牌与单行导航，不动其可视化引擎。

01设计原则

为什么改 —— 去掉「AI 生成感」，回到真正有设计的基本功。

移除 · Remove
  • 霓虹渐变背景、星云 mesh、点阵网格
  • 发光卡片、shimmer、glow 阴影
  • 渐变文字（bg-clip-text）
  • Fraunces + Inter 字体
采用 · Adopt
  • 平整表面 + 发丝级描边（暗）/ 柔和投影（亮）
  • 克制的单一领域强调色
  • 等宽元数据标签 + 大量留白
  • 双主题：观测台（暗）/ 手记（亮）
