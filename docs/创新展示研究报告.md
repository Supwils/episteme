## 一、核心论点（Thesis）

平台已经拥有**两样普通知识网站没有的资产**：

1. **沉浸式 R3F 宇宙**（`subjects/physics/` 的 Tier0→Tier7 三维下钻 + 自定义 GLSL）。
2. **一张 1076 节点、2297 条边的跨域知识图谱**（`subjects/knowledge-graph/`），其中 **1271 条边带有 `label` 字段，描述「两个概念为什么相连」——但这个 label 从未被渲染出来**。

> **结论：平台「不再像一本百科」的关键，不是「做更多交互文章」，而是让这张「观念之网」变得可导航、可读、能自我解释。** 差异化不在于把单篇文章做得更花哨，而在于**揭示跨学科的连接、建立直觉、制造「原来如此」的惊叹**。这条路线的最大优势是：**最难的部分（图引擎、力导向布局、最短路径、相机动画、wiki 链接解析）都已经建好，数据也已就位**——只需把它们「点亮」。

一个被研究反复印证的硬约束：**本仓库没有 MDX/JSX-in-prose 管线**，文章是纯 markdown 字符串由手写 `MarkdownRenderer` 渲染，块级交互通过既有 `interactive:` frontmatter 注册表注入。因此「把可调数字嵌进散文」这类 Bret Victor 式 reactive-document 想法属于**需要先投资解析器的低优先级轨道**，不应作为首发。

## 二、业界标杆与可借鉴模式（真实案例）

研究覆盖六个角度，以下为各角度最值得「偷师」的真实案例（均带 URL，2024–2026 现役）：

### 1. Explorable Explanations（可探索的解释）

- **Bret Victor — Explorable Explanations** (https://worrydream.com/ExplorableExplanations/)：reactive document（散文里的数字可拖拽）+「读者驱动的模型」。
- **Nicky Case — The Evolution of Trust / Parable of the Polygons / Adventures with Anxiety** (https://ncase.me/)：把抽象机制变成「读者亲手玩的小游戏」；视角反转（让你扮演「焦虑之狼」）。
- 启示：每个概念配一个「可操纵的小模型」远胜静态插图——但需 MDX 管线，列入二期。

### 2. 数据新闻与 Scrollytelling

- **The Pudding — sticky-graphic 范式** (https://pudding.cool/process/scrollytelling-sticky/)：左侧散文滚动、右侧粘性图随滚动变形。
- **NYT / Bloomberg Graphics**：注记直接画在可视化上、地图随滚动在时间维度动画。
- **Our World in Data** (https://ourworldindata.org)：「图先于文」——每个论断后面立刻跟上证明它的图。
- 启示：`地质年代`、`ScaleOfUniverse` 这类已有可视化可升级为「滚动即媒介」的章节。

### 3. 知识图谱与「连接」（本平台主场）

- **Quanta — Map of Mathematics** (https://mathmap.quantamagazine.org/)：图/文混合，从「数 / 形 / 变化」三颗种子progressive reveal，每个概念可点开。**这是本平台最该对标的形态。**
- **Wikiverse** (https://www.wikiverse.io/)：按链接密度把文章聚成可见的「星系」。
- **Obsidian 3D Graph + InfraNodus**：社区检测把簇推开，让结构一目了然。
- **cosmos.gl / Cosmograph** (https://cosmograph.app/)：GPU 力导向，百万节点实时重布局。

### 4. 沉浸式 WebGL / WebGPU

- **Maxime Heckel — Volumetric Lighting / TSL & WebGPU Field Guide** (https://blog.maximeheckel.com/)：在 `@react-three/postprocessing` 里合成 raymarch 体积光（god-ray）；GLSL→TSL 迁移到 WebGPU。
- **Codrops — GPGPU 粒子** (https://tympanus.net/codrops/2024/12/19/...)：FBO ping-pong 粒子系统。
- 启示：现有 Tier 场景可加「体积光后处理」「光谱恒星 shader」提升质感（中影响、二期）。

### 5. 生成式 / 构建期 AI 增强（无后端）

- **SemanticFinder** (https://do-me.github.io/SemanticFinder/)：用 Transformers.js（MiniLM）在**构建期**算好所有文章 embedding，发 `vectors.json`，前端纯余弦相似度做语义检索。
- **Observable Plot** (https://observablehq.com/plot/)：~10 行声明式从数组出图。
- 启示：**构建期跑一次 embedding** → 生成「你可能还会好奇」语义近邻 + 自动跨域边，零后端、零隐私问题。

### 6. 阅读体验与排版

- **Bartosz Ciechanowski — ciechanow.ski** (https://ciechanow.ski/mechanical-watch/)：每个口头概念后面立刻跟一个可拖拽的 widget——公认全网最好的交互解释。
- **gwern.net** (https://gwern.net/design)：边注（sidenotes）、链接悬停预览、可折叠「冰山」密度。
- **Wikipedia Page Previews** (https://www.mediawiki.org/wiki/Extension:Popups)：悬停链接出预览卡，解决「点链接就丢失上下文」。

## 四、旗舰功能（已实现 + 验证）：连接引擎（Connection Engine）

**为什么选它**：在整张清单里「单位工作量产出的洞察」最高——最难的部分都已建好，数据已就位；#1/#2/#4 三件事**共享同一套 UI 与 edge-label 渲染**，互相叠加。它正面攻击平台相对维基百科的差异点（揭示跨域连接），而不是去打磨某一篇文章。最具可演示性的一句话：**「任选两个观念，看见连接它们的真实思想之链。」**

### 4.1 现状诊断（实现前）

`subjects/knowledge-graph/components/PathFinder.tsx` 此前已能算最短路径，但：

- 只把路径渲染成**一串裸节点名**，**完全没有显示节点之间的「为什么相连」**——而那正是 `GraphEdge.label` 里躺着、从未露出的洞察；
- 有一个**域配色 bug**：硬编码只认 4 个域（physics/history/philosophy/life-science），其余 9 个域的路径节点全渲染成灰色；
- 找到路径后只有在「同时选中某节点」时画布才高亮该路径（一个边界 bug）。

### 4.2 本次落地（MVP，已浏览器验证通过）

改动 5 个文件，**完全不触碰有风险的 markdown 管线，影响半径锁定在 `/knowledge-graph` 路由**：

1. `data/thought-tours.ts`（新增）：7 条**经连通性与标签覆盖率验证**的策展「思想之线」（仅存路标 waypoint，运行时用最短路径缝合，因此每一跳都是图里真实的边、由 edge.label 自我旁白）：
   - **熵之线索**（物理→数学→经济→CS）：熵与时间之箭 → 概率论 → 博弈论 → 纳什 → 冯·诺伊曼 → 图灵
   - **理性的极限**（哲学→数学→CS）：苏格拉底 → … → 哥德尔不完备定理 → 可计算性
   - **心智之谜**（哲学→心理→经济）：笛卡尔 → … → 弗洛伊德 → 认知偏误 → 卡尼曼
   - **从大地到生命 / 从恒星到原子 / 演化的史诗 / 进化与社会**（达尔文→尼采→黑格尔→马克思→斯密）
2. `hooks/useGraphState.ts`：构建 `edgeLabelMap`（双向「a|b→label」）；新增 `handleTourSelect`（按存在的路标缝合最短路径，缺失则跳过，鲁棒）；修复 `highlightState` 使「找到路径即点亮画布」（不再依赖选中节点）。
3. `components/PathFinder.tsx`：改用真实 13 域配色；把路径渲染成**带「为什么相连」标注的思想谱系**——每两个概念之间显示其关系标签（如「尼采的权力意志与达尔文进化论的思想碰撞」「香农将熵形式化」），无标签时优雅降级为浅色斜体「相关联」；顶部加「思想之线」一键策展入口。
4. `components/GraphFilterBar.tsx` / `components/KnowledgeGraph.tsx`：把新 props（edgeLabelMap / tours / onTourSelect）透传下去。

附带修复一个**预先存在的 dev-only CSP bug**：`next.config.ts` 的 `worker-src 'self'` 在 Turbopack dev 下会拦截 blob: 力导向 worker（导致画布塌陷）；改为仅 dev 放行 `blob:`（生产仍严格，因为生产 worker 走同源 /\_next chunk）。

### 4.3 验证

- `pnpm typecheck` 通过（exit 0）。
- Playwright 浏览器实测：打开 `/knowledge-graph` → 点「连接引擎」→ 选「进化与社会」→ 正确渲染思想谱系「达尔文 → 尼采 → 黑格尔 → 卡尔·马克思 → 亚当·斯密」，每跳带关系标注、13 域配色正确、无标签处降级为「相关联」。

### 4.4 后续相位（同一套代码继续叠加）

- **二期**：手写 4–6 条更多策展 tour；语义缩放 LOD（星座→簇→细节）；相机沿路径「飞行」动画（`animateFocus` 已就绪）。
- **三期**：构建期跑一次 Transformers.js embedding → 喂给「语义近邻页脚」与图谱「语义布局」开关，让**概念结构**（而非仅引用结构）可见。

## 五、快速优化清单（可与旗舰并行的高 ROI 小改）

1. **画布边悬停时渲染 `GraphEdge.label`**：数据已存在、目前被丢弃；一天内即可把图从「装饰」升级为「会讲课」。
2. **wiki 链接悬停预览卡**：用原生 Popover API + 构建期 `slug→{title,excerpt,domain}` 旁车 JSON；阅读流最大的改善、JS 成本最低。
3. **仅跨域边过滤开关**：一个布尔值隐藏同域边，瞬间把全图变成纯跨学科地图。
4. **跨域边的渐变描边**（两端域色 `createLinearGradient`）：一眼可见的「学科交叉」。
5. **数学/物理文章的可折叠推导/证明块**（经 interactive 注册表）：gwern「冰山」密度，散文好读、严谨一键可达。
6. **阅读模式**：行宽 65–75 字、字号/行高加大、TOC 用 IntersectionObserver 高亮当前节——纯 CSS/JS，复用既有 `ArticleLayout`/`TableOfContents`。
7. **把文章页脚「相关条目」做成迷你子图**（复用 Canvas2D 渲染器），让「这篇文章在知识网中的邻域」可见而非罗列。

## 六、落地建议与风险

- **聚焦合规**：以上全部落在`聚焦方向.md`允许的「内容呈现 / 可视化 / 洞察」车道内，**不触碰**用户系统、收藏、学习路径、社交分享等禁区。
- **性能**：图谱与 3D 已懒加载；构建期 embedding 方案零运行时成本；wiki 预览走构建期旁车 JSON。共享首屏 JS 预算（gzip ≤ 180KB）不受影响。
- **可访问性**：所有动画尊重 `prefers-reduced-motion`（`animateFocus` 等已内建降级）。
- **风险**：edge.label 覆盖不均（已用「无标签降级为相关联」处理）；远距离最短路径可能很长或乏味（已有 `getNodesWithinHops` 可设上限，tour 用就近路标避免回环）。

> 一句话总结：把已经建好的「观念之网」点亮——让连接会自我解释、让思想之线可一键走过——平台就从「一个知识网站」变成「一台揭示人类知识如何彼此相连的机器」。
